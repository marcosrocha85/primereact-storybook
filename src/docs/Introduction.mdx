# Sakai Storybook

This project documents [Sakai React](https://github.com/primefaces/sakai-react) as a navigable design system: components, layout patterns, visual tokens, composition examples, and usage decisions.

## Goal

Create documentation that feels like a design system portal while staying connected to the PrimeReact ecosystem used by Sakai.

## What Is Ready

- Storybook 10 with React + Vite.
- English MDX documentation.
- PrimeReact, PrimeIcons, and PrimeFlex loaded globally.
- A custom visual theme for the manager and preview.
- A component-level documentation model under `Components/*`.
- A reproducible generator at `scripts/generate-component-stories.mjs`.
- `npm run sync:sakai` for downloading or updating the upstream repo in `vendor/sakai-react`.

You can find it at: [https://marcosrocha85.github.io/primereact-storybook/](https://marcosrocha85.github.io/primereact-storybook/)

## Next Milestones

1. Expand hand-written Controls for the most complex PrimeReact components.
2. Add accessibility notes per component.
3. Add usage guidelines and do/don't examples.
4. Publish the static Storybook with GitHub Pages.

## Sakai Components

The `Components/*` groups document the Sakai `uikit` demos at the component level without including application pages such as dashboard, landing, or auth.
